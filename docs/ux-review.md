# 家庭资产摘要 App UI/UX Review 报告

## 1. 评审结论

Family Asset Tracker 已经形成较完整的家庭资产月度复盘闭环：用户可以完成账号创建、配置成员与科目、录入月度快照、查看总览与趋势、管理历史快照，并通过备份导出降低数据丢失风险。

当前 UX 成熟度判断：**6.8/10，处于“可长期自用，但仍需降低理解成本与移动端录入成本”的阶段。**

主要优点是产品定位克制、核心流程清晰、金融指标已有解释意识、录入草稿和复制历史明显提升连续使用体验。主要短板是首次使用门槛仍偏高，月度录入在小屏和多科目场景下密度较大，总览与趋势仍偏“展示结果”而不是“解释变化”，部分危险操作和帮助提示还不够可感知、可追溯、可键盘访问。

## 2. 信息架构与用户路径评审

当前主导航包含：配置、总览、录入、趋势、备份、用户管理、账号。顶部流程条强调 `配置 -> 录入 -> 趋势`，这符合产品以月度快照为核心的主路径。

推荐的核心用户路径应明确为：

1. 首次使用：创建管理员账号 -> 配置成员 -> 套用或新增科目 -> 录入首期快照 -> 查看总览和趋势说明。
2. 每月更新：进入录入 -> 复制上一期 -> 调整日期、余额、收入、投资收益和备注 -> 保存 -> 查看趋势提示和总览。
3. 数据维护：配置页管理成员/科目启停 -> 录入页编辑历史周期 -> 备份页导出 JSON/CSV。
4. 风险恢复：备份导出 -> 导入恢复前确认覆盖风险 -> 导入后查看恢复结果。

主要问题：

- **P1：总览在导航中位于录入之前，但流程条不包含总览。** 当前用户保存后进入趋势，日常查看时又会回到总览，容易形成“总览到底是结果页还是首页”的认知断点。建议将信息架构明确为：`配置 -> 录入 -> 总览/趋势 -> 备份`，其中总览是最新一期结果页，趋势是长期分析页。
- **P1：备份是长期使用关键模块，但当前不在流程条中出现。** 对家庭资产数据而言，备份不是边缘功能。建议在完成首期录入后，给出“建议下载一次备份”的轻提示。
- **P2：用户管理、账号、备份与业务主流程混在同一导航层级。** 后续可将账号、用户管理、备份归入“设置/数据”区域，降低主导航认知负担。

## 3. 首次使用与新手引导

当前首次访问会创建管理员账号，配置页提供“初始化配置”“一键套用推荐模板”，并说明成员和科目可后续修改。这是正确方向。

主要问题：

- **P0：新手仍需要一次性理解成员、共同科目、科目类型、资产子类、投资收益、隐含支出等概念。** 对非财务敏感用户来说，当前说明偏简短，容易在首期录入前不知道如何选择归属和分类。
- **P1：推荐模板缺少预览。** 用户点击“一键套用推荐模板”前，不清楚会新增哪些科目、是否适合自己家庭结构。
- **P1：缺少示例模式或样例结果。** 用户需要先录入真实资产，才能看到总览和趋势价值，心理成本较高。
- **P2：首次配置成功后的下一步反馈较弱。** 当前只提示可进入录入，缺少“你已经配置了 X 位成员、Y 个科目”的完成感。

建议：

- 增加首次使用向导，分为三步：家庭成员、资产/负债科目、收入/支出科目。
- 在推荐模板按钮旁增加“查看模板内容”，展示将创建的科目名称、类型、归属默认值。
- 提供“使用示例数据体验”或“查看示例总览”，帮助用户先理解产品结果。
- 对“共同”增加解释：共同科目不会自动分摊到个人视角，适合房产、家庭存款、共同贷款等无法明确归属的项目。

## 4. 月度录入体验评审

当前录入页已具备良好基础：支持统计日期、备注、本期投资收益、按成员分组、共同分组、快速新增科目、本地草稿、提交前校验、历史查看/编辑/复制/删除。

主要问题：

- **P0：多科目场景下录入表单密度偏高。** 所有成员和共同科目一次性展开，家庭成员和科目数量增加后，用户需要长距离滚动，难以确认哪些字段已完成。
- **P0：移动端数字录入成本高。** 当前使用 `type="number"`，但缺少金额格式化、千分位展示、快速清零、复制上期差异提示、输入完成态。
- **P1：字段排序缺少“常用优先”和类别分组。** 当前按成员归属展示，但成员下资产、负债、收入、支出混排，用户录入时需要频繁切换心理模型。
- **P1：草稿状态反馈过轻。** “草稿已自动保存”显示在按钮旁，对长表单用户不够稳定，也未说明草稿保存范围和恢复条件。
- **P1：复制历史后的风险提示不足。** 复制上一期是高频好功能，但需要更明确提醒用户修改日期，并标出“继承自上一期”的字段。
- **P2：快速新增科目在录入页顶部，用户滚动到某个成员字段时发现缺科目，需要回到顶部操作。**

建议：

- 成员分组使用可折叠面板，默认展开最近编辑或有未完成字段的分组。
- 每个分组增加小计和完成状态，例如“张三：8 项，已修改 3 项”。
- 成员内按 `资产 -> 负债 -> 收入 -> 支出` 二级分组。
- 移动端字段改为更适合触控的单列卡片，每个字段提供大号金额输入、清零按钮、上期值参考。
- 保存按钮使用底部粘性操作区，显示“草稿已保存 / 有 X 项修改 / 保存并查看趋势”。
- 复制历史后在顶部使用醒目提示：“已复制 2026-05-31 数据，请确认统计日期和本期变化后保存。”

## 5. 总览体验评审

总览页展示统计视角、核心 KPI 和成员结构。指标覆盖净资产、本期变化、总资产、总负债、收入、投资收益、已记录支出、总支出、隐含支出，符合家庭资产复盘场景。

主要问题：

- **P0：核心指标视觉优先级不够强。** 净资产、本期变化、总资产、总负债、收入、支出等 KPI 使用同等卡片，用户不容易第一眼判断“这个月变好了还是变差了”。
- **P0：推算指标存在误读风险。** 总支出和隐含支出是根据净资产变化推算，不是原始流水事实。虽然已有帮助提示，但 `?` hover 在移动端和键盘环境不可用。
- **P1：个人视角规则不够前置。** 个人视角不分摊共同资产和共同负债，这会显著影响结果，应在切换视角附近明确说明。
- **P1：成员结构表在移动端可读性较弱。** 金额列较多，小屏横向阅读负担较大。
- **P2：缺少“本期变化解释”。** 用户看到净资产变化后，还需要自行判断来自收入、投资收益、负债下降还是支出异常。

建议：

- 总览顶部改为“本期结论卡”：净资产、较上期变化、变化方向、统计日期。
- 将推算指标标记为“估算”或“推算”，例如“推算总支出”“隐含支出估算”。
- 增加一段自动解释：`本期净资产增加 ¥X，主要由收入 ¥Y、投资收益 ¥Z、负债变化 ¥N 共同影响。`
- 成员结构移动端改为成员卡片，每张卡显示收入、资产、负债、净资产。
- 对异常值给出检查建议，而不仅是红绿颜色。

## 6. 趋势分析与图表可读性

趋势页已有统计范围配置、趋势提示、最新一期趋势摘要、净资产图、投资图、支出图、成员资产组成图、资产负债结构图和指标口径说明，整体分析框架较完整。

主要问题：

- **P0：趋势页信息量偏大，首屏任务不够聚焦。** 用户进入趋势页先看到统计范围配置，而不是趋势结论。统计范围是高级功能，不应压过分析结果。
- **P0：多轴投资图存在误读风险。** 投资市值、本期投资收益、本期净投入量级不同，放在同一图中容易让用户误判变化幅度。
- **P1：颜色语义不完全统一。** 图表颜色与页面中的上涨/下跌、资产/负债语义没有形成稳定映射。
- **P1：饼图不适合成员资产组成长期扩展。** 成员数量或共同科目增加后，饼图比较能力较弱，建议改为横向条形图或 100% 堆叠条。
- **P1：统计范围配置缺少收起策略。** 对普通用户来说，“排除科目”是高级分析功能，应默认折叠。
- **P2：图表缺少单位、区间说明和数据点解释。** 当前依赖 Chart.js tooltip，但页面静态信息不足。

建议：

- 趋势页顺序调整为：趋势摘要 -> 异常提示 -> 图表 -> 指标口径 -> 统计范围。
- 将统计范围放入 `高级设置` 折叠区。
- 投资分析拆成两张图：投资市值趋势、投资收益/净投入柱状对比。
- 成员资产组成改为横向条形图，显示金额和占比。
- 每张图增加简短说明，例如“负债以正数展示，便于与总资产比较”。
- 图表颜色建立固定规则：资产蓝/绿，负债红，收入绿，支出橙/红，估算指标紫或灰。

## 7. 视觉层级与布局

当前视觉风格符合产品文档中的“干净、克制、低干扰”：浅色背景、白色卡片、低饱和蓝色、细边框、轻阴影、统一圆角和基础状态色。

主要问题：

- **P1：卡片层级较一致，关键任务不够突出。** 配置、录入、总览、趋势中的主操作和辅助说明视觉权重接近。
- **P1：按钮体系需要进一步区分主次和危险级别。** 当前 `.btn-ghost.danger` 已有危险样式，但删除历史快照仍主要依赖原生 confirm。
- **P1：表格在移动端缺少专门样式。** 配置页和成员结构表都依赖表格，容易溢出或压缩。
- **P2：顶部导航在窄屏可能拥挤。** 当前使用 flex wrap，但业务导航、账号、退出混排后可扫描性下降。
- **P2：帮助提示使用 hover tooltip，不适合移动端。**

建议：

- 建立三层页面结构：页面目标卡、主任务卡、辅助/高级卡。
- 主按钮只保留一个，例如录入页主按钮为“保存并查看趋势”，其他操作使用次按钮。
- 危险操作使用自定义确认对话框，显示影响范围。
- 移动端将表格改为卡片列表，或至少提供横向滚动容器和冻结首列。
- 帮助说明改为可点击的说明抽屉、popover 或展开说明，不依赖 hover。

## 8. 状态反馈评审

已覆盖的状态：

- 空状态：总览无配置、无数据；趋势无数据；配置无成员/科目；录入无历史周期。
- 加载/保存状态：配置保存按钮显示“保存中...”，模板显示“应用中...”。
- 错误状态：表单错误、图表加载失败、导入错误。
- 成功状态：配置保存成功、导入成功、密码更新成功。
- 危险确认：历史快照删除 confirm、备份导入 confirm 和 checkbox。

主要问题：

- **P0：危险操作确认不够具体。** 删除历史快照和导入覆盖会影响金融历史数据，应显示被影响的周期、数量和不可恢复后果。
- **P1：成功反馈较分散。** 有些反馈在状态文本，有些跳转后显示，有些只改变按钮文字，缺少统一模式。
- **P1：加载状态没有覆盖趋势图渲染和页面级等待。**
- **P1：空状态缺少行动导向。** 例如趋势无数据时只提示去录入，没有解释“至少两期后才能看到变化趋势”。
- **P2：错误信息缺少修复建议。** 例如金额非法、导入失败、隐含支出异常，都可以补充“下一步检查什么”。

建议：

- 建立统一状态组件：成功、信息、警告、错误、危险。
- 删除历史快照确认文案包含日期和说明：`将删除 2026-05-31 快照及所有明细，趋势图会重新计算。`
- 导入恢复增加导入前摘要：成员数、科目数、快照期数、最早/最新日期。
- 趋势空状态明确说明：`已保存 1 期，保存第 2 期后可计算净资产变化、总支出和投资净投入。`
- 草稿恢复后提供“查看恢复内容 / 清除草稿 / 继续编辑”。

## 9. 可访问性评审

当前基础较好：页面设置了 `lang="zh-CN"`，表单大多使用 label 包裹输入，焦点样式清晰，颜色对比总体可接受。

主要问题：

- **P0：hover 帮助提示无法被键盘和移动端稳定访问。** `.help-tip` 是 span + CSS hover，不是按钮，也没有键盘焦点。
- **P1：原生 confirm/alert 对复杂操作说明不足，且不可定制焦点管理。**
- **P1：颜色承担了部分金融含义。** 上涨/下跌、警告/成功不应只靠颜色区分。
- **P1：动态状态文本未使用 aria-live。** 保存成功、草稿保存、错误提示对屏幕阅读器不够友好。
- **P2：图表缺少文本替代。** Canvas 图表需要提供摘要文本或数据表入口。
- **P2：导航缺少当前页标识。** 视觉上流程条有 active，但主导航链接没有明确 `aria-current`。

建议：

- 将帮助提示改为 `<button type="button">`，支持 focus、Enter、Space，并提供 `aria-expanded`。
- 状态提示容器添加 `role="status"` 或 `aria-live="polite"`；错误提示使用 `role="alert"`。
- 图表下方提供“查看数据表”或简短文字摘要。
- 主导航当前页增加 `aria-current="page"`。
- 危险操作弹窗打开后焦点进入弹窗，关闭后回到触发按钮。

## 10. P0/P1/P2 优先级清单

### P0：必须优先解决

- 降低推算指标误读风险：将总支出、隐含支出、投资净投入明确标记为估算/推算，并补充适用条件。
- 优化月度录入长表单：支持分组折叠、保存状态固定展示、移动端单列输入。
- 改造危险操作确认：删除快照、导入覆盖必须显示影响范围和二次确认。
- 让帮助提示可点击、可键盘访问、可移动端使用。
- 趋势页首屏优先展示分析结论，而不是统计范围配置。

### P1：下一版重点体验提升

- 增加首次使用向导和推荐模板预览。
- 总览页增加本期变化解释卡。
- 录入页增加上期值参考、复制历史来源提示和字段完成状态。
- 趋势图拆分高误读图表，统一颜色语义。
- 配置页和成员结构表移动端改为卡片化展示。
- 备份导入增加导入前摘要和自动导出当前备份提醒。
- 个人视角切换处明确说明共同科目不分摊。

### P2：后续增强

- 示例数据/演示模式。
- 月度复盘摘要自动生成。
- 财务健康指标：负债率、储蓄率、现金覆盖月数、投资占比。
- 科目排序、常用项置顶、自定义分组。
- 目标管理：净资产目标、现金储备目标、负债下降目标。
- 图表数据表、CSV 快速入口和年度对比视图。

## 11. 下一版 UX 改进建议

建议下一版主题为：**“低摩擦月度复盘版”**。

推荐范围：

1. 首次使用向导：成员、科目、推荐模板预览、共同科目解释。
2. 录入页重构：折叠分组、上期值参考、移动端金额输入、底部粘性保存条。
3. 总览结论卡：突出净资产、本期变化、变化归因和异常检查项。
4. 趋势页重排：摘要优先，统计范围折叠为高级设置。
5. 状态与危险操作统一：自定义确认对话框、导入摘要、删除影响说明。
6. 可访问性补强：可键盘访问帮助提示、aria-live 状态、图表文本摘要。

## 12. 给 PM 的建议

- 明确定义“总览”和“趋势”的关系：总览回答“最新一期怎么样”，趋势回答“长期变化如何”。
- 补充个人视角规则：共同资产/负债是否永远不分摊，还是未来支持可选分摊。
- 明确推算指标命名策略：是否使用“推算总支出”“隐含支出估算”等更保守表达。
- 定义推荐模板内容和适用人群，避免模板过度暗示用户必须按某种资产结构管理。
- 确认移动端是否是一等使用场景。如果用户主要每月在手机上录入，录入页需要优先重构。

## 13. 给 Dev 的建议

- 将帮助提示组件化，替换 hover-only span。
- 为表单状态、alert、confirm dialog 建立统一组件和样式。
- 录入页按成员和科目类型输出结构化分组，便于折叠和移动端卡片化。
- 为趋势图增加文本摘要和数据表 fallback。
- 将统计范围配置折叠，默认不占用趋势页首屏。
- 删除和导入操作返回影响范围数据，用于确认弹窗展示。

## 14. 给 QA 的建议

重点测试以下 UX 验收场景：

- 首次用户从无配置到完成首期录入，是否能不查文档理解成员、共同科目和科目类型。
- 手机宽度下录入 3 位成员、30 个科目，是否能高效完成输入、保存和草稿恢复。
- 只有 1 期数据时，总览和趋势是否明确说明哪些指标不适用。
- 隐含支出为负、净资产大幅波动、投资收益为 0 但市值变化大时，提示是否可理解、可行动。
- 删除历史快照和导入备份前，用户是否能明确知道影响范围。
- 键盘用户是否能访问导航、表单、帮助提示、保存、删除和导入操作。
- 移动端是否可以阅读所有表格或替代表格卡片。
- 图表加载失败时是否有明确错误提示，不出现空白区域。

## 15. 设计验收标准

下一版可按以下标准验收：

- 首次使用时，用户可以在 5 分钟内完成成员和基础科目配置，并能说明“共同科目”的含义。
- 推荐模板应用前可以查看将新增的科目列表，应用后不会覆盖已有科目。
- 录入页在移动端 390px 宽度下无横向溢出，主保存按钮始终容易找到。
- 用户复制上一期后，页面明确显示复制来源，并提示修改统计日期。
- 任一推算指标旁均有可点击说明，说明包含公式、适用条件和误读风险。
- 趋势页首屏优先展示最新一期摘要和异常提示，统计范围默认不干扰普通查看。
- 删除历史快照确认中必须展示周期日期和“将删除明细并重算趋势”的说明。
- 导入恢复前必须展示覆盖风险，并建议先下载当前备份。
- 所有状态反馈通过统一样式展示，成功、错误、警告、危险有明确视觉区分和文本说明。
- 帮助提示、弹窗、表单错误和保存状态支持键盘访问和屏幕阅读器感知。
- 图表至少提供文字摘要，用户不看图也能理解最新一期关键变化。

## 16. 最终判断

这个项目的 UX 基础是健康的：产品没有过度扩张，核心路径围绕“家庭资产月度快照”展开，配置、录入、趋势和备份已经形成真实闭环。

下一阶段最重要的不是增加更多功能，而是让用户更容易开始、更快完成每月录入、更少误解金融指标，并在危险操作前更有把握。完成 P0 和 P1 后，产品体验会从“技术用户可长期自用”提升到“普通家庭用户也能稳定试用”。
