# 关键缺口开发计划

本文档对应产品评价中 `4.2 关键缺口` 的四个优先建设项，不修改原评价报告，用于指导下一阶段实现。

## 1. 登录、访问控制与隐私保护

目标是避免家庭资产数据在公网部署时裸露。第一阶段采用单用户管理员账号，首次访问时创建管理员，之后通过邮箱和密码登录；所有业务页面和接口均需登录后访问，健康检查和静态资源保持可访问。第二阶段再考虑多用户账号、密码修改、操作审计和敏感数值隐藏。

验收标准：

- 未登录访问总览、配置、录入、趋势、备份等页面会跳转到登录页。
- 首次访问时可以创建唯一管理员账号。
- 登录成功后写入 HttpOnly Cookie，会话可保持一段时间。
- 用户可以主动退出登录。
- 系统不依赖全站共享访问密码作为正式登录方式。

## 2. 数据导入、导出和备份

目标是保证用户可以把数据带走并在故障后恢复。第一阶段提供完整 JSON 导出和快照 CSV 导出；导入以 JSON 为准，覆盖恢复成员、科目、周期和快照值。第二阶段再增加自动定期备份、备份版本兼容说明和导入前差异预览。

验收标准：

- 用户可以在备份页面下载完整 JSON 备份。
- 用户可以下载适合表格查看的快照 CSV。
- 用户可以粘贴 JSON 备份并导入恢复。
- 导入过程使用事务，失败时不写入部分数据。

## 3. 历史快照查看、编辑、复制、删除

目标是让长期月度数据可管理。第一阶段在录入页列出已有周期，支持查看编辑、复制为新一期和删除。编辑时表单回填历史值；复制时回填历史值但日期默认进入新周期；删除需要二次确认。第二阶段再增加历史对比、变更记录和批量管理。

验收标准：

- 已记录周期可从录入页直接进入编辑。
- 复制已有周期可以减少重复录入。
- 删除周期会同步删除对应快照值。
- 历史编辑不会丢失已停用科目的历史值。

## 4. 录入草稿和保存前校验

目标是降低误填和中断风险。第一阶段使用浏览器本地草稿自动保存录入表单，提交前校验日期和数值字段；服务端再次校验，避免无效数据入库。第二阶段再增加服务端草稿、异常波动提醒和更细的业务规则。

验收标准：

- 录入页修改后会自动保存本地草稿。
- 页面重新打开时可以恢复未提交草稿。
- 保存前会阻止空日期、非法数字和明显不完整的数据。
- 服务端对提交数据进行同等基础校验。

## 推荐实施顺序

1. 先实现访问保护，降低公网部署风险。
2. 再实现导出和导入，确保后续调整前已有可恢复备份。
3. 然后实现历史快照编辑、复制和删除，解决长期维护问题。
4. 最后补录入草稿与保存前校验，提升连续使用体验。
