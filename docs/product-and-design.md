# 家庭资产更新 App：功能与设计理念

## 产品定位

家庭资产更新 App 是一个面向家庭长期资产管理的轻量 Web 应用。它不是传统记账工具，而是以“月度资产快照”为核心，帮助用户持续记录家庭成员、资产、负债、收入和投资表现，并从中生成净资产、支出、投资与负债趋势。

产品目标是：

- 降低每月更新家庭资产的录入成本
- 支持家庭与个人两个统计视角
- 允许用户自定义家庭成员和科目
- 用统一口径展示净资产、投资、收入、支出和负债变化
- 保持界面干净、克制、可维护，适合长期使用

## 核心使用流程

应用围绕三个阶段组织：

1. 配置
   - 添加家庭成员
   - 配置资产、负债、收入、支出科目
   - 支持推荐模板快速初始化
   - 支持后续持续维护，不是一次性设置

2. 录入
   - 每月录入一次资产快照
   - 录入当前各项资产、负债、收入等数据
   - 可在录入页快速新增缺失科目
   - 保存后进入趋势页查看结果

3. 趋势
   - 展示净资产变化
   - 展示投资市值、投资收益和净投入
   - 展示总支出、已记录支出和隐含支出
   - 展示资产与负债结构变化

顶部流程条以 `配置 -> 录入 -> 趋势` 引导用户完成主要任务。

## 功能模块

### 1. 家庭成员管理

家庭成员是资产与负债归属的基础维度。

支持能力：

- 新增成员
- 查看成员
- 编辑成员名称
- 启用或停用成员
- 删除成员
- 批量保存成员修改

删除保护：

- 如果成员已经有关联科目，保存时不会删除该成员
- 系统会跳过这类删除操作，并保留其他合法修改
- 用户可以通过“停用”隐藏不再使用的成员

设计原则：

- 删除是破坏性操作，必须保护历史数据
- 停用优先于删除
- 成员管理与科目管理采用一致的编辑模式

### 2. 科目管理

科目是每月录入的字段来源。用户可以按家庭实际情况配置科目，例如招商银行、农业银行、房贷、工资收入等。

支持能力：

- 新增科目
- 查看科目
- 搜索科目
- 显示或隐藏已停用科目
- 编辑科目名称
- 修改科目类型
- 修改资产子类
- 修改归属成员
- 启用或停用科目
- 标记删除
- 批量保存所有修改

删除保护：

- 如果科目已有历史录入数据，保存时不会删除该科目
- 系统会跳过该项删除，并继续保存其他修改
- 用户可以停用已有历史的科目，避免破坏历史趋势

### 3. 科目分类体系

一级分类保持克制，只包含四类：

- 资产
- 负债
- 收入
- 支出

资产子类用于进一步区分资产结构：

- 现金/存款
- 投资
- 公积金
- 其他

设计理由：

- 股票、基金等不作为一级分类，而属于资产中的投资子类
- 投资收益不是科目分类，而是每期录入的指标
- 收入和支出作为流量类数据保留，用于解释净资产变化
- 支出可以手动记录，也可以由系统推算未分类部分

### 4. 月度录入

月度录入页用于记录某一统计日的家庭资产快照。

用户录入：

- 统计日期
- 本期备注
- 本期投资收益
- 各成员名下科目金额
- 共同科目金额

支持能力：

- 按成员分组展示录入字段
- 共同科目单独分组
- 缺少科目时可在页面内快速新增
- 新增后返回录入页，新字段自动出现

设计原则：

- 录入页应专注于“本期数据”
- 配置入口应可达，但不应打断录入主流程
- 快速新增科目用于解决录入时发现字段缺失的问题

### 5. 总览

总览页展示当前最新一期的关键指标。

核心指标：

- 家庭净资产
- 本期变化
- 总资产
- 总负债
- 本期收入
- 本期投资收益
- 已记录支出
- 总支出
- 隐含支出

支出口径：

- 总支出 = 本期收入 + 本期投资收益 - 本期净资产变化
- 隐含支出 = 总支出 - 已记录支出

这使用户可以只录入收入、资产、负债和投资收益，系统自动推算未被手动归类的支出。

### 6. 家庭与个人视角

应用支持两种统计视角：

- 家庭视角：汇总全部家庭成员和共同科目
- 个人视角：只统计选定成员名下科目

当前规则：

- 共同科目不会自动分摊到个人视角
- 这样可以避免错误归属
- 如果未来需要，可增加共同资产/负债的分摊规则

### 7. 趋势分析

趋势页以图表展示长期变化。

当前图表：

- 净资产趋势
- 投资市值、投资收益、本期净投入
- 总支出、隐含支出、已记录支出
- 总资产与总负债结构

技术与稳定性设计：

- Chart.js 使用本地依赖，不依赖外部 CDN
- 图表 canvas 使用固定高度，避免浏览器计算异常
- 图表加载失败时显示明确错误信息，而不是空白区域

## 数据与计算理念

### 快照优先

产品以“每期快照”为核心，而不是逐笔流水。

好处：

- 录入成本低
- 更适合家庭资产月度复盘
- 可直接计算净资产趋势
- 不强迫用户记录所有消费明细

### 存量与流量分离

资产和负债是存量，描述某个时点的余额。

收入和支出是流量，描述某个周期内发生的金额。

产品通过这两类数据建立解释关系：

- 资产负债决定净资产
- 收入、投资收益和净资产变化共同解释支出

### 投资收益与净投入

投资类资产属于资产。

本期投资收益单独录入，用于计算投资净投入：

```text
本期净投入 = 本期投资市值 - 上期投资市值 - 本期投资收益
```

这样可以区分：

- 投资市值因为市场波动变化
- 投资市值因为用户追加或赎回资金变化

## UI 设计理念

### 1. 干净、克制、低干扰

界面采用浅色背景、卡片式结构和低饱和色彩。重点是让用户快速完成月度更新，而不是制造视觉噪音。

设计原则：

- 大面积留白
- 低饱和品牌色
- 细边框与轻阴影
- 明确但不夸张的状态色

### 2. 主次清晰

每个页面只突出一个主要任务：

- 配置页：管理成员和科目
- 录入页：录入本期数据
- 趋势页：查看变化趋势

次要操作收起或放入次级区域，例如录入页的快速新增科目、配置页的新增科目面板。

### 3. 编辑模式统一

成员和科目管理都采用统一编辑模式：

- 默认浏览
- 点击编辑进入批量编辑
- 表格内直接修改
- 底部保存条统一提交

这样避免多选、单行删除和编辑按钮混杂造成歧义。

### 4. 保护历史数据

涉及历史记录的数据不轻易删除。

产品原则：

- 可停用
- 可隐藏
- 删除前保护历史关联
- 保存时跳过不允许删除的项

这保证长期趋势数据不会因为误删科目或成员而损坏。

### 5. 解释不打扰

复杂概念不直接堆在页面上，而是通过提示说明。

例如：

- 总支出
- 隐含支出
- 本期投资收益

需要解释的地方使用帮助提示，而不是在标题里写大量括号说明。

## 部署设计

当前项目使用：

- Node.js / Express
- EJS 页面渲染
- PostgreSQL 数据库
- Chart.js 图表
- Render Web Service
- Render Postgres

部署理念：

- 使用免费额度优先
- 依赖尽量少
- 后端和页面一体化，便于小型产品维护
- 通过 `render.yaml` 支持 Render Blueprint 部署
